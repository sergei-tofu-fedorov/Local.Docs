# Step 5.4: Anonymous (Signature-Auth) Users

## Problem

Signature-auth users (Account-Id + Signature, see [Authentication](../../../../Backend/HowTo/Authentication.md) — Scenario 2) have **no Bearer token** and **no MasterUser**. `UpsertJobCommandHandler` always calls `_workerService.GetTeam()` → `ITofuAuthApiClient.GetTenantUsersAsync()`, which requires a Bearer token to forward to Tofu.Auth. Under signature auth this call **fails**.

These users have no team and never send worker assignments — jobs just need to work without the team fetch.

## Solution: Guard on `MasterUserId`

`MasterUserId` (on `CommandBase`) is null for signature auth, non-null for bearer auth. `GetTeam()` accepts `string? masterUserId` and returns `Team.Empty` when null — no Auth API call.

Convention: **any service method requiring Auth API access accepts `string? masterUserId` and returns a safe default when null.**

### Changes

`IJobWorkerService.GetTeam()` — added `masterUserId` parameter:

```csharp
Task<Team> GetTeam(string accountId, string? masterUserId, CancellationToken ct);
```

`JobWorkerService.GetTeam()` — returns `Team.Empty` when null:

```csharp
if (masterUserId == null)
{
    _logger.LogDebug("No master user for account {AccountId}, returning empty team", accountId);
    return Team.Empty;
}
```

`UpsertJobCommandHandler` — passes `command.MasterUserId`:

```csharp
var team = await _workerService.GetTeam(command.AccountId, command.MasterUserId, ct);
```

### Not affected

- **`WorkerUpdateVisitStatusCommandHandler`** — `WorkerController` throws if `MasterUserId` is null; signature-auth users never reach it.
- **`Job.UpdateVisits()`** — already handles empty team gracefully (silently clears worker assignments).
