# Step 5.0: Separate MemberAccounts Array

> Separate admin and non-admin account memberships in `MasterUser`.
> `OwnedAccounts` — admin/owner only. New `MemberAccounts` embedded
> array with dedicated `MemberAccount` class for workers and future roles.
> `AllAccountIds` computed property for read paths.

## Data Model

### `MasterUser` — changes

**File:** `Invoices.Core/Models/MasterUser.cs`

```csharp
public required List<OwnedAccount> OwnedAccounts { get; set; }

[BsonIgnoreIfNull]
public List<MemberAccount>? MemberAccounts { get; set; }

/// <summary> All accounts (owned + member) for read paths. </summary>
[BsonIgnore]
public IReadOnlyList<string> AllAccountIds =>
    OwnedAccounts.Select(x => x.AccountId)
        .Concat((MemberAccounts ?? []).Select(x => x.AccountId)).ToList();
```

Key decisions:
- **Nullable** `List<MemberAccount>?` — old documents don't have the field
- **`[BsonIgnoreIfNull]`** — prevents writing `null` to MongoDB on `Upsert`,
  keeps field missing (not `null`) so `$push` works on old documents
- **Not initialized** in `CreateUser` — stays `null` until first member added
- **`AllAccountIds`** returns `IReadOnlyList<string>` (account IDs only),
  not full objects — read paths only need IDs

### `MemberAccount` — new class

**File:** `Invoices.Core/Models/MasterUser.cs`

```csharp
public sealed class MemberAccount
{
    public required string AccountId { get; init; }
    public required OwnedAccountMeta MemberAccountMeta { get; init; }

    [BsonIgnoreIfNull]
    public string? TenantRole { get; init; }
}
```

Separate from `OwnedAccount` — different semantics, own meta field name.
Reuses `OwnedAccountMeta` (contains `AssignedBy`).

### `OwnedAccount` — simplified

`TenantRole` field **removed** — `OwnedAccounts` array is strictly for
admins/owners, role is implicit.

### `TryAddAccount` — no change

Checks only `OwnedAccounts` — this is the admin account creation path.
Member accounts are added through `AddOrUpdateInvitedAccount`.

### MongoDB indexes

**Unique index on `OwnedAccounts.AccountId`:**

```
Key:     OwnedAccounts.AccountId ASC
Unique:  true
Partial: (none — TenantRole removed from OwnedAccount)
```

Guarantees one admin/owner per account across the entire collection.
Previously had partial filter on `TenantRole == null` — no longer
needed since `OwnedAccounts` only contains admin entries.

**No index on `MemberAccounts`** — multiple users can be members of
the same account. Intra-document uniqueness enforced at application
level (`AllAccountIds` check before `$push`).

## Edge Cases

### Existing documents without MemberAccounts field

MongoDB returns `null` for missing fields.

- **Read:** `AllAccountIds` handles `MemberAccounts ?? []`
- **`$push`:** Auto-creates array when field is **missing** in document
- **`$push` on explicit `null`:** Would fail — but `[BsonIgnoreIfNull]`
  prevents `null` from being written on `Upsert`, keeping field missing
- **`$pull`:** No-op on missing field

### Intra-document uniqueness for MemberAccounts

MongoDB unique indexes enforce uniqueness **between documents**, not
within a single document's array. `$addToSet` compares entire objects,
not individual fields.

**Solution:** Application-level check in `AddOrUpdateInvitedAccount` —
reads user, checks `AllAccountIds`, throws before `$push`.
