# Step 5.2: Worker Endpoints Update — Implementation Plan

> Worker ID propagation, assignment checks, and in-progress blocking
> for the worker visit endpoints.

## Motivation

The Worker API (`/api/worker/visits`) is functional but has data
isolation gaps that must be fixed before enabling RBAC:

| Gap | Severity | Details |
|-----|----------|---------|
| Worker ID not passed to read queries | High | `GetWorkerVisitsPagedQuery`, `GetWorkerVisitsStatsQuery`, `GetWorkerVisitDetailsQuery` receive `AccountId` but no `WorkerId` — data isolation depends on repository implementation |
| Visit status update missing checks | High | `TryUpdateVisitStatusByWorker` does not check `AssignedWorkerId` (any worker can update any visit) and does not block when another visit is already in progress |
---

## Plan

Ensure every worker query explicitly filters by the authenticated
worker's ID, not just by `AccountId`.

### 1.5 Move assignment check into `Job.TryUpdateVisitStatusByWorker`

**Current state:** The domain method receives `WorkerId` (via
`workerName` lookup) but never checks `visit.AssignedWorkerId`.
This gap should be fixed inside the aggregate.

#### Updated method signature

**File:** `Jobs/Jobs.Domain/Models/Job.cs`

Add `string workerId` parameter so the method can validate assignment:

```csharp
public VisitStatusUpdateResult TryUpdateVisitStatusByWorker(
    Guid visitId,
    string workerId,
    VisitStatus newStatus,
    string? workerName,
    DateTimeOffset occurredAt)
```

#### Full validation flow inside the method

```csharp
public VisitStatusUpdateResult TryUpdateVisitStatusByWorker(
    Guid visitId, string workerId, VisitStatus newStatus,
    string? workerName, DateTimeOffset occurredAt)
{
    var visit = Visits?.FirstOrDefault(v => v.Id == visitId);
    if (visit == null)
        return VisitStatusUpdateResult.NotFound();

    // 1. Assignment check — worker must be assigned to this visit
    if (visit.AssignedWorkerId != workerId)
        return VisitStatusUpdateResult.Forbidden();

    if (visit.Status == newStatus)
        return VisitStatusUpdateResult.NoChange();

    // 2. In-progress blocking — only one visit can be in progress
    if (newStatus == VisitStatus.InProgress)
    {
        var inProgressVisit = Visits?.FirstOrDefault(
            v => v.Id != visitId && v.Status == VisitStatus.InProgress);
        if (inProgressVisit != null)
            return VisitStatusUpdateResult.VisitStatusChangeBlocked(inProgressVisit.Id);
    }

    var previousStatus = visit.Status;
    visit.UpdateStatus(newStatus);

    RaiseDomainEvent(JobDomainEvent.VisitStatusChanged(
        Id, visitId, previousStatus, newStatus, workerName));
    RefreshComputedFields(occurredAt);
    return VisitStatusUpdateResult.Success();
}
```

#### `VisitUpdateError` (discriminated union for domain errors)

**File:** `Jobs/Jobs.Domain/Models/VisitStatusUpdateResult.cs`

```csharp
public abstract record VisitUpdateError
{
    public record NotFound : VisitUpdateError;
    public record Forbidden : VisitUpdateError;
    public record Blocked(Guid BlockingVisitId) : VisitUpdateError;
}
```

Domain method returns `Result<bool, VisitUpdateError>` — `true` if status
changed, `false` if same status (no-op). Errors are mapped to exceptions
in the handler.

#### Handler — map results to errors

**File:** `Jobs/Jobs.Application/Worker/Commands/WorkerUpdateVisitStatusCommandHandler.cs`

```csharp
var result = job.TryUpdateVisitStatusByWorker(
    cmd.VisitId, cmd.WorkerId, cmd.Status.ToDomain(), workerName, now);

// Errors throw exceptions → middleware maps to HTTP status codes
if (result.IsFailure) { /* NotFound, Forbidden, Blocked → throw */ }

// On success, return updated visit data
var visit = job.Visits!.First(v => v.Id == cmd.VisitId);
return WorkerMappings.ToVisitUpdateResult(visit, job.Version, now);
```

**Success response** (`200`): Returns the updated visit and job version.

```json
{
  "visit": {
    "id": "...", "jobId": "...", "dateTime": "...",
    "assignedWorkerId": "...", "status": "inProgress",
    "isOverdue": false, "statusChangedAt": "..."
  },
  "version": 2
}
```

**Error responses** (thrown as exceptions → middleware):
- `NotFound` → `404` — visit doesn't exist
- `Forbidden` → `403` — visit exists but not assigned to this worker
- `VisitStatusChangeBlocked` → `409` with `visitStatusChangeBlocked` error code
  and `blockingVisitId` — another visit is already in progress

#### `WorkerAccessDeniedException` (global worker exception)

A reusable exception for any worker endpoint where the worker
attempts to access or modify a resource they are not assigned to.
Currently used only for write operations (visit status update).
Read queries continue to filter by `workerId` at the repository
level (unassigned visits simply don't appear in results).

**File:** `Invoices.Core/Exceptions/WorkerAccessDeniedException.cs` (new)

```csharp
/// <summary> Thrown when a worker attempts to access a resource they are not assigned to. </summary>
public class WorkerAccessDeniedException : Exception
{
    public string WorkerId { get; }
    public string Resource { get; }

    public WorkerAccessDeniedException(string workerId, string resource)
        : base($"Worker '{workerId}' does not have access to {resource}")
    {
        WorkerId = workerId;
        Resource = resource;
    }
}
```

**File:** `Invoices.Api/Middleware/ApiExceptionHandlingMiddleware.cs`

```csharp
new Map<WorkerAccessDeniedException>(
    HttpStatusCode.Forbidden, ErrorCode.Forbidden, false,
    e => e.Message,
    logLevel: LogLevel.Warning),
```

No new `ErrorCode` needed — reuses existing `ErrorCode.Forbidden`.

#### `VisitStatusChangeBlockedException` and error code

**File:** `Invoices.Core/Exceptions/VisitStatusChangeBlockedException.cs` (new)

```csharp
/// <summary> Thrown when a visit status change is blocked because another visit is already in progress. </summary>
public class VisitStatusChangeBlockedException : Exception
{
    public Guid VisitId { get; }
    public Guid BlockingVisitId { get; }
    public VisitStatusChangeBlockedException(Guid visitId, Guid blockingVisitId)
        : base($"Visit '{visitId}' status change blocked by in-progress visit '{blockingVisitId}'")
    {
        VisitId = visitId;
        BlockingVisitId = blockingVisitId;
    }
}
```

**File:** `Invoices.Api/Middleware/ErrorCode.cs`

```csharp
[EnumMember(Value = "visitStatusChangeBlocked")]
VisitStatusChangeBlocked,
```

**File:** `Invoices.Api/Middleware/ApiExceptionHandlingMiddleware.cs`

```csharp
new Map<VisitStatusChangeBlockedException>(
    null, (int)HttpStatusCode.Conflict, ErrorCode.VisitStatusChangeBlocked, false,
    e => e.Message,
    e => new { e.BlockingVisitId },
    logLevel: LogLevel.Warning),
```

**Client behavior:** On receiving `409` with `visitStatusChangeBlocked`,
the client should use `blockingVisitId` to navigate the worker to
the in-progress visit so they can complete it first.
