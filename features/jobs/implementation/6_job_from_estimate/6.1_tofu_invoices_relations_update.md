```csharp
private static EstimateEvent BuildEstimateEvent(
    EstimateJobCreatedDomainEvent domainEvent,
    string accountId, string estimateId, string? masterUserId,
    int estimateVersion, ActorType actorType)
{
    var payload = EventJsonSerializer.SerializePayload(new JobCreatedPayload
    {
        JobId = domainEvent.JobId,
        JobNumber = domainEvent.JobNumber
    });

    return EstimateEvent.Create(accountId, estimateId, masterUserId,
        EventType.JobCreated, actorType, payload, estimateVersion,
        domainEvent.OccurredAt);
}
```

## 2. Handle jobNumber in Add RPCs

When Invoices.Backend saves an invoice or estimate with a `JobId`, it passes `jobNumber` in the gRPC Add request. Tofu.Invoices uses this to generate timeline events without needing to look up the job.

**Sync scenario:** If the job hasn't been synced yet, Invoices.Backend cannot find it and sends an empty `jobNumber`. Tofu.Invoices still generates timeline events with an empty `jobNumber` — the event is always recorded regardless, and the client handles the missing job number in the display text.

### 2.1 Estimate Add — `estimateJobCreated` event

When the estimate's `JobId` changes from null/empty to a non-null value during an Add call, Tofu.Invoices generates an `estimateJobCreated` timeline event. If `jobNumber` is empty (sync scenario), the event is still created with an empty `jobNumber`.

**Proto change:** add `optional google.protobuf.StringValue job_number = 4;` to `Estimates.V1.AddRequest`.

This covers the "Save Estimate with JobId" flow — the event is generated inside the existing Add RPC, not via `LinkJobToEstimate`.

### 2.2 Invoice Add — `invoiceCreatedFromJob` event

When a new invoice with a `JobId` is saved via the Add call, Tofu.Invoices generates an `invoiceCreatedFromJob` timeline event. If `jobNumber` is empty (sync scenario), the event is still created with an empty `jobNumber`.

**Proto change:** add `optional google.protobuf.StringValue job_number = 4;` to `Invoices.V1.AddRequest`.

## 3. Handle invoice deletion — clear InvoiceId on Estimate

When an invoice is deleted via the existing Delete RPC and that invoice has an `EstimateId`, Tofu.Invoices should clear `InvoiceId` on the linked estimate.

This runs inside the existing invoice Delete handler — no new RPC needed.

**Logic:**
```
if deleted invoice has EstimateId:
    load estimate by EstimateId
    if estimate exists and estimate.InvoiceId == deleted invoice ID:
        set estimate.InvoiceId = null
        save estimate
```

This ensures the estimate no longer references a deleted invoice.
