# Step 6.3: Create Job From Estimate

Covers requirements 1 (Status gate), 2 (Currency), and 3 (Activity events) from the overview, plus a refactor of estimate services.

## Core flow (already implemented)

The handler (`CreateJobFromEstimateCommandHandler`) already:
- Fetches the estimate via `GetEstimateWithItems()` which returns the estimate model and its items converted to `JobItem[]`.
- Uses the **estimate's client** (`estimate.Client.CatalogId`) as the job's client. The job title defaults to the client name.
- **Copies all line items** from the estimate to the job via `job.UpdateItems(jobItems)` — each `InvoiceItem` is mapped to a `JobItem` (name, description, unit price, quantity, discount, tax, catalog ID, item type).
- Links the estimate to the job (`job.TryAddEstimateLink`) and writes the `JobId` back to the estimate via gRPC.

## 0. Refactor: consolidate estimate services

Move `GetEstimateWithItems` and `GetItemsFromEstimate` from `JobItemsService` into `JobEstimateService`, then delete `JobItemsService`.

`JobEstimateService` already owns estimate-related operations (linking, unlinking). Fetching an estimate with its items belongs in the same place.

**Steps:**

1. **`IJobEstimateService`** — add methods from `IJobItemsService`:
   - `Task<(Estimate, JobItem[])> GetEstimateWithItems(string accountId, string estimateId, CancellationToken ct)`
   - `Task<JobItem[]> GetItemsFromEstimate(string accountId, string estimateId, CancellationToken ct)`

2. **`JobEstimateService`** — move the implementation (including the private `ToJobItem` mapper) from `JobItemsService`.

3. **`CreateJobFromEstimateCommandHandler`** — replace `IJobItemsService` dependency with `IJobEstimateService` (already injected for `TryLinkEstimate`). Remove the `_jobItemsService` field.

4. **`CopyEstimateItemsToJobCommandHandler`** — replace `IJobItemsService` dependency with `IJobEstimateService`.

5. **Delete** `IJobItemsService.cs` and `JobItemsService.cs` from `Services\Items\`.

6. **`JobsModule.cs`** — remove `services.AddScoped<IJobItemsService, JobItemsService>()` registration.

## 3. Activity events

### 3.1 Job-side: `jobCreated`

A single `jobCreated` event fires for **all** new jobs — whether created manually or from an estimate. No separate `jobCreatedFromEstimate` event type.

The existing `JobDomainEvent.Created` with `EmptyPayload` already handles this. No new event types, payloads, or enum values needed on the job side.

### 3.2 Estimate-side: `estimateJobCreated` — two paths

The `estimateJobCreated` event is generated by **Tofu.Invoices** whenever an estimate's `JobId` changes from null to non-null. This covers two scenarios:

| Flow | Who triggers | Mechanism |
|------|-------------|-----------|
| Create Job From Estimate | `CreateJobFromEstimateCommandHandler` | `LinkJobToEstimate` RPC (§4 below) |
| Save Estimate with JobId | Client PUT /estimates | Tofu.Invoices Add RPC detects `JobId` changed null → value ([6.1 §2.1](6.1_tofu_invoices_link_job.md)) |

**Client-side for Create Job From Estimate:**
1. Add `LinkJobToEstimate` method to `IEstimatesGateway` interface
2. Implement client call in `Tofu.Invoices\EstimatesGateway.cs`
3. Rewrite `JobEstimateService.TryLinkEstimate` — replace fetch-mutate-save with the single gRPC call:

```csharp
public async Task TryLinkEstimate(Job job, string estimateId,
    JobLinkContext context, CancellationToken ct)
{
    await _estimatesGateway.LinkJobToEstimate(new LinkJobToEstimateRequest
    {
        AccountId = context.AccountId,
        EstimateId = estimateId,
        Job = new JobInfo
        {
            Id = job.Id.ToString(),
            Number = job.Number
        },
        OccurredAtMs = context.OccurredAt?.ToUnixTimeMilliseconds(),
        MasterUserId = context.MasterUserId
    }, ct);
}
```

4. Wrap call in try/catch — failure should not fail job creation

**Order of operations in the Create Job From Estimate handler:**
1. Fetch estimate, validate status = Approved
2. Create job, set currency (existing `jobCreated` domain event fires automatically)
3. Save job + job events to DB
4. Call `TryLinkEstimate(job, estimateId, linkContext, ct)` — Tofu.Invoices sets `JobId` and creates `estimateJobCreated` event (since `JobId` changes from null to non-null)
5. Client makes a separate call to transition the estimate to Done

### 4.3 Aggregate method

Add a `LinkJob` method to `EnrichedEstimate`. Domain events must be registered inside the aggregate (`RegisterDomainEvent` is private), following the same pattern as `SetStatus` and `SetMailStatus`.

**File:** `Tofu.Invoices.Domain\Models\Estimate\EnrichedEstimate.cs`

```csharp
public void LinkJob(string jobId, string jobNumber, DateTimeOffset? occurredAt)
{
    var wasUnlinked = string.IsNullOrEmpty(Entity.JobId);

    Entity.JobId = jobId;

    if (wasUnlinked)
    {
        RegisterDomainEvent(new EstimateJobCreatedDomainEvent
        {
            JobId = jobId,
            JobNumber = jobNumber,
            OccurredAt = occurredAt ?? DateTimeOffset.UtcNow,
            DocNumber = Entity.Number
        });
    }
}
```

**Responsibilities:**
- Sets `Entity.JobId` unconditionally (write-once; only called when linking)
- If `JobId` changed from null/empty to non-null: registers `EstimateJobCreatedDomainEvent`
- If `JobId` was already set (re-link / idempotent call): no event
- Does **not** transition status to Done — the client handles that via a separate call

### 4.4 Command handler

The handler delegates all domain logic to `EnrichedEstimate.LinkJob()` and handles only persistence.

**File:** `Tofu.Invoices.Domain\Commands\LinkJobToEstimate\LinkJobToEstimateCommandHandler.cs`

```csharp
public sealed class LinkJobToEstimateCommandHandler
    : ICommandHandler<LinkJobToEstimateCommand, LinkJobToEstimateResult>
{
    private readonly IEstimatesRepository _repository;
    private readonly IEstimateEventsRepository _eventsRepository;
    private readonly IEstimateEventsFactory _eventsFactory;

    public LinkJobToEstimateCommandHandler(
        IEstimatesRepository repository,
        IEstimateEventsRepository eventsRepository,
        IEstimateEventsFactory eventsFactory)
    {
        _repository = repository;
        _eventsRepository = eventsRepository;
        _eventsFactory = eventsFactory;
    }

    public async Task<LinkJobToEstimateResult> Handle(
        LinkJobToEstimateCommand command, CancellationToken token)
    {
        var estimate = await _repository.Get(command.AccountId, command.EstimateId, token);
        var enriched = estimate.ToEnrichedEstimate();

        enriched.LinkJob(command.JobId, command.JobNumber, command.OccurredAt);

        var saved = await _repository.InsertOrUpdate(enriched.Entity, token);

        // Persist timeline events (job created)
        var events = _eventsFactory.CreateEstimateEvents(enriched, command.MasterUserId, ActorType.System);
        if (events.Count != 0)
            await _eventsRepository.Add(events, token);

        return new LinkJobToEstimateResult(saved);
    }
}
```

**Key behavior:**
- `JobId` was null/empty → sets `JobId`, creates `JobCreated` timeline event
- `JobId` was already set → sets `JobId` only, no duplicate events (idempotent)
- Does **not** transition status — the client makes a separate call to set estimate status to Done

### 4.5 Wire up gRPC service

**File:** `Tofu.Invoices.Api\Grpc\V1\EstimatesService.cs`

```csharp
public override async Task<LinkJobToEstimateResponse> LinkJobToEstimate(
    LinkJobToEstimateRequest request, ServerCallContext context)
{
    var command = new LinkJobToEstimateCommand(
        AccountId: request.AccountId,
        EstimateId: request.EstimateId,
        JobId: request.Job.Id,
        JobNumber: request.Job.Number,
        OccurredAt: request.OccurredAtMs > 0
            ? DateTimeOffset.FromUnixTimeMilliseconds(request.OccurredAtMs)
            : null,
        MasterUserId: request.MasterUserId);

    var result = await _linkJobHandler.Handle(command, context.CancellationToken);

    return new LinkJobToEstimateResponse
    {
        Estimate = EstimatesServiceMapping.MapToEstimateObj(result.Estimate)
    };
}
```

Inject `ICommandHandler<LinkJobToEstimateCommand, LinkJobToEstimateResult>` in the constructor.
