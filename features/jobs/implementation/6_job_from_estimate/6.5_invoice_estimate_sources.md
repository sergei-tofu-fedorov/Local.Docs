# Step 6.5: Add Source Enum to Invoice and Estimate

## Context

Invoices and estimates can be created standalone, from an estimate, or from a job. The creation source is currently inferred at creation time to emit the correct domain event, but is never persisted. We need to store it as an enum field so clients can read it without inferring from `JobId`/`EstimateId` presence.

**Goal:** Add a `Source` enum field to both entities, computed once at creation, stored in MongoDB, and exposed read-only in gRPC responses.

### 2. Entity Properties

### 3. Set Source During Creation

#### InvoiceSource -- `EnrichedInvoice.Create()` (line 28)

Source is determined in the existing three-branch `if/else if/else` block (lines 44-68) that already picks the domain event. Add one assignment per branch:

```
if (!string.IsNullOrWhiteSpace(invoice.JobId))          // line 44
{
    enrichedInvoice.Entity.Source = InvoiceSource.Job;   // <-- ADD
    RegisterDomainEvent(new InvoiceCreatedFromJobDomainEvent { ... });
}
else if (!string.IsNullOrWhiteSpace(invoice.EstimateId)) // line 54
{
    enrichedInvoice.Entity.Source = InvoiceSource.Estimate; // <-- ADD
    RegisterDomainEvent(new InvoiceCreatedFromEstimateDomainEvent { ... });
}
else                                                     // line 64
{
    enrichedInvoice.Entity.Source = InvoiceSource.None;  // <-- ADD
    RegisterDomainEvent(new InvoiceCreatedDomainEvent { ... });
}
```

Source is set once during `Create` and never modified by `Update`.

| Condition | Source value | Domain event |
|-----------|-------------|--------------|
| `JobId` is present | `Job` | `InvoiceCreatedFromJobDomainEvent` |
| `EstimateId` is present (no JobId) | `Estimate` | `InvoiceCreatedFromEstimateDomainEvent` |
| Neither present | `None` | `InvoiceCreatedDomainEvent` |
| Existing record (field missing in DB) | `null` | (no event -- not present in MongoDB) |

#### EstimateSource -- `EnrichedEstimate.Create()` (line 28) and `Update()` (line 150)

`Create` initialises a bare `Entity` with defaults (line 33-41), then calls `Update()` (line 45). Source is set in `Create()` **before** calling `Update()`, alongside the creation event (see section 7e):

```
// In Create(), after registering the creation event and before calling Update():
if (!string.IsNullOrWhiteSpace(estimate.JobId))
{
    enrichedEstimate.Entity.Source = EstimateSource.Job;     // <-- ADD
    enrichedEstimate.RegisterDomainEvent(new EstimateCreatedFromJobDomainEvent { ... });
}
else
{
    enrichedEstimate.Entity.Source = EstimateSource.None;    // <-- ADD
    enrichedEstimate.RegisterDomainEvent(new EstimateCreatedDomainEvent { ... });
}
enrichedEstimate.Update(estimate, logger, occurredAt, jobNumber);
```

Inside `Update` there is an existing block that detects when a JobId is assigned for the first time. This must only emit `EstimateJobCreatedDomainEvent` for **existing** estimates that get a job linked later -- not during initial creation (where `Create()` already emitted `EstimateCreatedFromJobDomainEvent`). Guard with `Entity.Source != EstimateSource.Job`. **Source is never set in `Update()`** -- it is immutable after `Create()`:

```
if (!string.IsNullOrWhiteSpace(updated.JobId))
{
    var hadNoJob = string.IsNullOrEmpty(Entity.JobId);
    Entity.JobId = updated.JobId;

    if (hadNoJob && Entity.Source != EstimateSource.Job) // <-- guard
    {
        RegisterDomainEvent(new EstimateJobCreatedDomainEvent { ... });
    }
}
```

When `Create()` sets `Source = Job` before calling `Update()`, the guard prevents a duplicate event. When an existing estimate (Source is `None`, `Unknown`, or `null`) gets a job linked via `Update()`, the guard passes and the event fires. Source remains unchanged -- it reflects the **creation** origin only.

| Scenario | Source value | Where set | Event |
|----------|-------------|-----------|-------|
| New estimate with `JobId` | `Job` | `Create()` | `EstimateCreatedFromJobDomainEvent` |
| New estimate standalone | `None` | `Create()` | `EstimateCreatedDomainEvent` |
| Existing estimate linked to job later | unchanged (`None`/`Unknown`/`null`) | `Update()` `hadNoJob` block (event only, Source not changed) | `EstimateJobCreatedDomainEvent` |
| Existing estimate already has job, updated again | unchanged | skipped (`hadNoJob` is false) | none |
| Existing record (field missing in DB) | `null` | (not present in MongoDB) | none |

### 7. New Estimate Timeline Events: `estimateCreated` / `estimateCreatedFromJob`

Currently `EnrichedEstimate.Create()` emits a single `EstimateCreatedDomainEvent` for all estimates. In `EstimateEventsFactory` this is mapped to `EventType.StatusChanged` with payload `{ from: null, to: "draft" }` -- piggybacking on the status-changed event type.

Per `Timeline Events.tsv`, estimates need two dedicated creation events (mirroring how invoices already have `invoiceCreated` / `invoiceCreatedFromEstimate` / `invoiceCreatedFromJob`):

| Event type | Condition | Payload | Text in estimate timeline |
|------------|-----------|---------|--------------------------|
| `estimateCreated` | Standalone estimate (no JobId) | `{ "docNumber": "ES001" }` | You created this estimate |
| `estimateCreatedFromJob` | Estimate created with JobId | `{ "jobId": "guid", "jobNumber": "J001", "docNumber": "ES001" }` | You created this estimate from Job #J001 |

#### 7e. Split `EnrichedEstimate.Create()` event emission

Replace the single `EstimateCreatedDomainEvent` in `Create()` (line 43-44) with a branch. Source is set here too (see section 3). Both happen **before** calling `Update()`:

```csharp
if (!string.IsNullOrWhiteSpace(estimate.JobId))
{
    enrichedEstimate.Entity.Source = EstimateSource.Job;
    enrichedEstimate.RegisterDomainEvent(new EstimateCreatedFromJobDomainEvent
    {
        OccurredAt = occurredAt ?? DateTimeOffset.UtcNow,
        DocNumber = estimate.Number,
        JobId = estimate.JobId,
        JobNumber = jobNumber ?? string.Empty
    });
}
else
{
    enrichedEstimate.Entity.Source = EstimateSource.None;
    enrichedEstimate.RegisterDomainEvent(new EstimateCreatedDomainEvent
    {
        OccurredAt = occurredAt ?? DateTimeOffset.UtcNow,
        DocNumber = estimate.Number
    });
}

enrichedEstimate.Update(estimate, logger, occurredAt, jobNumber);
```

The `Update()` `hadNoJob` block is guarded with `Entity.Source != EstimateSource.Job` so it only fires `EstimateJobCreatedDomainEvent` for existing estimates getting a job linked later (see section 3). Source is never modified in `Update()` -- it is immutable after creation.

#### 7f. Update `EstimateEventsFactory`

Add two new `BuildEstimateEvent` overloads and update the `switch`:

```csharp
// In the switch:
EstimateCreatedDomainEvent created => BuildEstimateCreatedEvent(
    created, entity.AccountId, entity.Id, masterUserId, entity.Version, actorType),
EstimateCreatedFromJobDomainEvent createdFromJob => BuildEstimateCreatedFromJobEvent(
    createdFromJob, entity.AccountId, entity.Id, masterUserId, entity.Version, actorType),

// New builder -- standalone:
private static EstimateEvent BuildEstimateCreatedEvent(...)
{
    var payload = new EstimateCreatedPayload { DocNumber = domainEvent.DocNumber };
    // EventType.Created
}

// New builder -- from job:
private static EstimateEvent BuildEstimateCreatedFromJobEvent(...)
{
    var payload = new EstimateCreatedFromJobPayload
    {
        DocNumber = domainEvent.DocNumber,
        JobId = domainEvent.JobId,
        JobNumber = domainEvent.JobNumber
    };
    // EventType.CreatedFromJob
}
```

The old `BuildEstimateEvent(EstimateCreatedDomainEvent)` that mapped to `EventType.StatusChanged` with `{ from: null, to: "draft" }` is removed.

#### 7h. Backward compatibility

Old estimates that were created before this change have an `estimateStatusChanged` event with `{ from: null, to: "draft" }` as their creation event. Clients should treat `statusChanged(null -> draft)` as a creation event when no `estimateCreated` / `estimateCreatedFromJob` event exists for the same entity (see TSV condition: "no other estimate created events").

## What Does NOT Change

- **AddRequest proto messages** -- Source is server-computed; client value is ignored
- **Command handlers** -- they pass Invoice/Estimate to enriched Create which sets Source
- **MongoDB** -- no migration needed; `[BsonIgnoreIfNull]` means the field is absent for old documents and reads as `null`
- **Existing records** -- remain `Source = null`; gRPC mapping converts `null` to proto `Unknown`
