# Step 6.9: API Contract Changes Summary

Consolidated list of all API contract changes introduced by steps 6.1–6.8 (excluding activity/timeline event types). Use this as a migration checklist for client apps.

### API Reference Docs

- [Invoices API Reference](../../../../Backend/Api/INVOICES_API_REFERENCE.md)
- [Estimates API Reference](../../../../Backend/Api/ESTIMATES_API_REFERENCE.md)
- [Jobs API Reference](../../../../Backend/Api/JOBS_API_REFERENCE.md)

## Invoice DTO

**Endpoint**: `PUT /api/v1/invoices`, `GET /api/v1/invoices/{id}`, etc.

| Field | Type | Direction | Change | Notes |
|-------|------|-----------|--------|-------|
| `jobId` | `string?` | Read/Write | **Existing** | Client sends on create to link invoice to job |
| `estimateId` | `string?` | Read/Write | **Existing** | Client sends when creating invoice from estimate |
| `source` | `string?` | Read-only | **New** | Creation origin: `"none"`, `"estimate"`, `"job"`. Omitted from JSON for legacy records (pre-Source). Server-computed, never sent by client |

**Source values** (`InvoiceSourceDto`):

| Value | Numeric | Condition |
|-------|---------|-----------|
| `"none"` | 1 | Standalone invoice |
| `"estimate"` | 2 | Created from estimate (has `estimateId`, may also have `jobId`) |
| `"job"` | 3 | Created from job (has `jobId`) |
| *(omitted)* | — | Legacy record (field absent in response) |

## Estimate DTO

**Endpoint**: `PUT /api/v3/estimates`, `GET /api/v3/estimates/{id}`, etc.

| Field | Type | Direction | Change | Notes |
|-------|------|-----------|--------|-------|
| `jobId` | `string?` | Read/Write | **Existing** | Client sends on create to link estimate to job |
| `invoiceId` | `string?` | Read-only | **New** | ID of invoice linked through the shared job. Server-computed (set by Tofu.Invoices when invoice is linked via job). Do NOT send on upsert |
| `source` | `string?` | Read-only | **New** | Creation origin: `"none"`, `"job"`. Omitted for legacy records. Server-computed |

**Source values** (`EstimateSourceDto`):

| Value | Numeric | Condition |
|-------|---------|-----------|
| `"none"` | 1 | Standalone estimate |
| `"job"` | 2 | Created from a job |
| *(omitted)* | — | Legacy record (field absent in response) |

## Job DTO

**Endpoint**: `PUT /api/v3/jobs`, `GET /api/v3/jobs/{id}`, `GET /api/v3/jobs/paged`, etc.

| Field | Type | Direction | Change | Notes |
|-------|------|-----------|--------|-------|
| `invoiceId` | `string?` | Read-only | **Existing** | Derived from `Job.Relations.Invoices[0]`. Previously accepted on upsert (deprecated in 6.2, removed in 6.6) |
| `estimateId` | `string?` | Read-only | **Existing** | Derived from `Job.Relations.Estimates[0]`. Previously accepted on upsert (deprecated in 6.2, removed in 6.6) |
| `currencyCode` | `string?` | Read-only | **New** | Currency code (e.g. `"USD"`). Defaults to `"USD"`. Copied from estimate's currency when job is created via `POST /jobs/from-estimate`. Not accepted on upsert |

**`invoiceId` / `estimateId` on upsert**: These fields were accepted on `PUT /api/v3/jobs` in older versions but are now **removed** from the upsert request. Relation linking is handled by sending `jobId` on the invoice/estimate upsert instead.

## Job Paged List DTO (items in `GET /api/v3/jobs/paged`)

| Field | Type | Change | Notes |
|-------|------|--------|-------|
| `currencyCode` | `string?` | **Existing** | Already present in summary items. Display currency logic updated: uses `Job.CurrencyCode` (from estimate) instead of static `"USD"` default when no invoice is linked |

## Onboarding DTO

**Endpoint**: `GET /api/v3/onboarding/status`, `POST /api/v3/onboarding/skip-step`, `POST /api/v3/onboarding/dismiss-modal`

| Field | Type | Direction | Change | Notes |
|-------|------|-----------|--------|-------|
| `isFromGenerator` | `bool` | Read-only | **New** | Whether the user came from the invoice generator. Clients use this to decide whether to show the "convert to job" suggestion modal |
| `modalDismissals` | `ModalDismissalDto[]` | Read-only | **New** | Array of dismiss counts per modal type |
| `modalDismissals[].modalType` | `string` | Read-only | **New** | Modal type enum. Current value: `"estimateToJob"` |
| `modalDismissals[].dismissCount` | `int` | Read-only | **New** | Number of times the user dismissed this modal. Client hides modal when count reaches threshold (e.g. 3) |

### New Endpoint: Dismiss Modal

**`POST /api/v3/onboarding/dismiss-modal`**

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `modal` | `string` | Yes | Modal type to dismiss. Current value: `"estimateToJob"` |

**Response**: Full `OnboardingStatusDto` with updated `modalDismissals`.

**Modal type values** (`ModalTypeDto`):

| Value | Numeric | Description |
|-------|---------|-------------|
| `"estimateToJob"` | 1 | "Convert estimate to job" suggestion modal |

## Error Codes (new)

| Code | Endpoint | Condition |
|------|----------|-----------|
| `jobAlreadyHasInvoice` | `PUT /api/v1/invoices` | Invoice has `jobId` but the job already has a different invoice linked |
| `jobAlreadyHasEstimate` | `PUT /api/v3/estimates` | Estimate has `jobId` but the job already has a different estimate linked |
| `clientArchived` | `PUT /api/v3/jobs` | Creating a job for an archived client (existing) |
