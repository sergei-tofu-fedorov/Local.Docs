# Step 6.4: Onboarding Updates

Covers requirements 6 (isFromGenerator exposure) and 7 (modal dismissal counter) from the overview.

## Scenario

When a user tries to convert an estimate directly to an invoice, the client shows a modal suggesting they convert to a **job** instead (the recommended workflow: Estimate → Job → Invoice). After the user dismisses this modal 3 times, the client stops showing it.

To support this, the backend needs to:
1. **Expose `isFromGenerator`** — clients use this flag to decide whether to show the job suggestion modal (only relevant for non-generator users).
2. **Provide a generic modal dismissal counter** — a reusable mechanism on the `Onboarding` entity that tracks how many times a user dismissed a specific modal type, so the client knows when to stop showing it.

## 1. Expose `isFromGenerator` in the API response

`IsFromGenerator` is already stored on the `Onboarding` entity and returned in the core `OnboardingStatus` model, but the API DTO (`OnboardingStatusDto`) does not include it.

### 1.1 Add field to DTO

**File:** `Invoices.Api\Dto\OnboardingStatusDto.cs`

```csharp
/// <summary> Whether the user came from the invoice generator. </summary>
public bool IsFromGenerator { get; set; }
```

### 1.2 Map it

**File:** `Invoices.Api\Models\OnboardingMapping.cs`

In `ToDto()`, add:

```csharp
IsFromGenerator = status.IsFromGenerator,
```

No backend logic changes — the flag is already computed and available in `OnboardingStatus`.

### 2.7 Add dismiss endpoint

**File:** `Invoices.Api\Controllers\OnboardingController.cs`

```csharp
/// <summary> Increment the dismiss counter for a modal. </summary>
[HttpPost("dismiss-modal")]
public async Task<ActionResult<OnboardingStatusDto>> DismissModal(
    [FromBody] DismissModalRequest request, CancellationToken ct)
{
    var status = await _onboardingService.DismissModal(AccountId, request.Modal, ct);
    return status.ToDto();
}
```

### 2.8 Add service method

**File:** `Invoices.Core\Services\IOnboardingService.cs`

```csharp
Task<OnboardingStatus> DismissModal(string accountId, ModalType modalType, CancellationToken ct = default);
```

**File:** `Invoices.Implementation.Services\Onboarding\OnboardingService.cs`

```csharp
public async Task<OnboardingStatus> DismissModal(string accountId, ModalType modalType, CancellationToken ct)
{
    var onboarding = await _onboardingRepository.FindByAccountId(accountId);
    if (onboarding == null)
        onboarding = InitializeNewOnboarding(accountId, fromGenerator: false);

    var dismissal = onboarding.ModalDismissals.FirstOrDefault(m => m.ModalType == modalType);
    if (dismissal == null)
    {
        dismissal = new ModalDismissal { ModalType = modalType };
        onboarding.ModalDismissals.Add(dismissal);
    }

    dismissal.DismissCount++;
    await _onboardingRepository.Upsert(onboarding);

    return await BuildOnboardingStatus(onboarding, accountId, ct);
}
```

The endpoint returns the full updated `OnboardingStatus` so the client can use the response directly (same pattern as `MarkStepAsSkipped`). To add a new dismissable modal in the future, just add a value to `ModalType`.
