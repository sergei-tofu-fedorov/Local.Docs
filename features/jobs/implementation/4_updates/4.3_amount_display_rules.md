# 4.3 Amount Display Rules

Stats and list amounts follow completion-based display logic.

---

## Plan: Amounts Restructuring

### Changes in This Stage

1. **Replace `GetInvoiceDisplayAmount()` with `GetDisplayAmount()`** — new method adds completion-aware logic: shows scope total for non-completed jobs, invoice amounts only when completed + invoice linked
2. **Add `GetDisplayCurrencyCode()`** — returns invoice currency when available, falls back to `Job.DefaultCurrencyCode` (`USD`)
3. **Add new DTO fields** — `TotalScope`, `DisplayAmount`, and `ManualStatus` added to contract and API response DTOs
4. **Use `GetDisplayCurrencyCode()`** — replaces direct `Amounts.CurrencyCode` access in mapping

> **Not in this stage:** Adding `CurrencyCode` to the `Job` table. For now, `Job.CurrencyCode` uses a hardcoded default (`USD`). A future stage will persist it per-job and allow setting it from job items or invoice data.

### Design Decisions

**No database migrations.** These changes are purely in application-layer logic (C# methods and DTO mappings). No new columns or schema changes are introduced — the jobs feature is still evolving rapidly, so we avoid unnecessary migrations until the schema stabilizes.

**TotalScope is recalculated on every request, not stored as a column.** `Job.CalculateTotalScope()` iterates `Job.Items` and sums `unitPrice * quantity`. A typical job has a small number of items (single digits), so this is a near-instant in-memory operation. Denormalizing it into a column would add migration overhead and a sync obligation for no measurable performance gain.

### TotalScope Access

`Job.CalculateTotalScope()` lives on the `Job` entity only. `JobSummaryView` accesses it via its `Job` navigation property (join). No denormalization needed — the summary already loads `Job` for other fields.

### Currency Source

Currency comes from the linked invoice when available, otherwise falls back to `Job.DefaultCurrencyCode` (`USD`).

| Scenario | Currency Source |
|----------|---------------|
| Invoice linked | `Amounts.CurrencyCode` |
| No invoice | `Job.DefaultCurrencyCode` (`USD`) |

---

## Amount Sources

| Source | Calculation | Currency |
|--------|-------------|----------|
| **Scope Total** | Sum of job items (`unitPrice * quantity`) | `Amounts.CurrencyCode` or default `USD` |
| **Invoice Amount** | From linked invoice (`totalAmount` or `totalDue`) | `Amounts.CurrencyCode` |

---

## Display Logic

| Manual Status | Invoice State | Amount Source | Currency Source |
|---------------|---------------|---------------|-----------------|
| Not Completed | Any (or none) | Scope Total | `Amounts.CurrencyCode` or default `USD` |
| Completed | No invoice | Scope Total | default `USD` |
| Completed | Invoice unpaid | `invoice.totalDue` | `Amounts.CurrencyCode` |
| Completed | Invoice paid | `invoice.totalAmount` | `Amounts.CurrencyCode` |

**Key rule:** Invoice amounts only apply when **both**: (1) job is manually marked **Completed**, AND (2) invoice is linked. Otherwise, always use Scope Total. Currency always comes from `Amounts.CurrencyCode` when available, falling back to `Job.DefaultCurrencyCode` (`USD`).

## Implementation

### Domain Layer

**File**: `Jobs.Domain/Models/InvoiceAmounts.cs`

Denormalized amount object stored in `JobSummaryView` for list display efficiency:

```csharp
    public decimal? TaxAmount { get; set; }
    public decimal? TotalAmount { get; set; }
    public decimal? TotalDue { get; set; }
}
```

**File**: `Jobs.Domain/Models/JobSummaryView.cs`

`TotalScope` is **not** stored as a column — it is calculated on the fly from `Job.CalculateTotalScope()`.

```csharp
public InvoiceAmounts Amounts { get; set; } = new();

/// <summary> Returns the amount to display based on completion status and invoice state. </summary>
public decimal GetDisplayAmount()
{
    var totalScope = Job.CalculateTotalScope();

    // If not completed, always show Scope Total
    if (Job.ManualStatus != JobManualStatus.Completed)
        return totalScope;

    // If completed but no invoice, show Scope Total
    if (string.IsNullOrEmpty(InvoiceId))
        return totalScope;

    // If completed with invoice: TotalAmount for paid, TotalDue for unpaid
    return InvoiceStatus == JobInvoiceStatus.Paid
        ? Amounts.TotalAmount ?? totalScope
        : Amounts.TotalDue ?? Amounts.TotalAmount ?? totalScope;
}
```

> **Note**: `GetDisplayAmount()` calculates scope total from `Job.Items` on each call — no denormalized column needed. Frontend should use `DisplayAmount` directly.

**File**: `Jobs.Domain/Models/Job.cs`

```csharp
public const string DefaultCurrencyCode = "USD";

public decimal CalculateTotalScope() =>
    Items?.Sum(item => item.UnitPrice * item.Quantity) ?? 0;
```

> `DefaultCurrencyCode` is used as fallback when no invoice currency is available.

### Application Layer (BFF Mapping)

**File**: `Jobs.Application/JobsMappings.cs`

Maps domain model to contract DTO. `TotalScope` is calculated from `Job`, not read from a view column:

```csharp
public static JobSummaryItemDto ToJobSummaryItemDto(this JobSummaryView summary)
{
    var job = summary.Job;

    return new JobSummaryItemDto
    {
        Id = job.Id,
        Number = job.Number,
        Title = job.Title,
        Status = ToContract(summary.EffectiveStatus),
        InvoiceStatus = ToContract(summary.InvoiceStatus),
        ManualStatus = ToContract(job.ManualStatus),
        CreatedAt = job.CreatedAt,
        CurrencyCode = summary.GetDisplayCurrencyCode(),
        TotalScope = job.CalculateTotalScope(),
        TotalDue = summary.Amounts.TotalDue ?? 0,
        TotalAmount = summary.Amounts.TotalAmount ?? 0,
        DisplayAmount = summary.GetDisplayAmount(),
        CompletionTime = job.CompletionTime,
        InvoiceId = job.Relations.Invoices.FirstOrDefault()?.Id,
        ActionsRequired = summary.GetActionsRequired().Select(ToContract).ToList(),
        NextVisitDate = summary.NextVisitDate
    };
}
```

**Key**: Backend computes `DisplayAmount` based on completion status and invoice state. Frontend uses this directly.

**New fields:**

### Display Currency

The currency returned for stats depends on which amount source `GetDisplayAmount()` picks:

```csharp
/// <summary> Returns the currency code matching the display amount source. </summary>
public string GetDisplayCurrencyCode()
{
    // Use invoice currency when available, otherwise fall back to default
    return !string.IsNullOrEmpty(Amounts.CurrencyCode)
        ? Amounts.CurrencyCode
        : Job.DefaultCurrencyCode;
}
```

| Scenario | Currency Source |
|----------|----------------|
| Invoice linked | `Amounts.CurrencyCode` |
| No invoice | `Job.DefaultCurrencyCode` (`USD`) |

### Stats Aggregation Flow

Stats use `GetDisplayAmount()` and `GetDisplayCurrencyCode()` per job, then aggregate:

```
JobSummaryView[] (all for account)
        │
        ▼
Per job: (GetDisplayAmount(), GetDisplayCurrencyCode())
        │
        ▼
GroupBy(currency) per status   ← In-memory, O(n)
        │
        ▼
Sum DisplayAmount per currency per status
        │
        ▼
AmountByCurrencyDto[] (multi-currency totals)
```
