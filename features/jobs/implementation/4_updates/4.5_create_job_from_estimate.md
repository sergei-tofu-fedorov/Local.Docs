# 4.5 Create Job from Estimate

## Overview

The "create job from estimate" endpoint already exists (`POST /jobs/from-estimate`). This task adds:
1. New estimate activity event when job is created (in Tofu.Invoices)
2. Invoice link in estimate response when job has invoice (resolved at API layer)

## Current State

| Aspect | Current Implementation |
|--------|------------------------|
| Endpoint | `POST /jobs/from-estimate?estimateId={id}` (v3.0) |
| Bidirectional link | `Estimate.JobId` <-> `Job.Relations.Estimates` |
| Communication | gRPC via `EstimatesService` to Tofu.Invoices |
| Estimate events | `EstimateStatusChanged`, `EstimateEmailStatusChanged` (no job-created event) |

---

## Part 1: New Estimate Event "Job Created"

### Solution

Create event in Tofu.Invoices via new gRPC call. This ensures the event is stored with the estimate and visible in estimate timeline queries.

### Flow

```
Invoices.Backend                         Tofu.Invoices
      |                                        |
      |  1. CreateJobFromEstimate              |
      |     (creates job, links estimate)      |
      |                                        |
      |  2. NotifyJobCreated(estimateId,       |
      |     jobId, jobNumber)                  |
      |--------------------------------------->|
      |                                        |
      |                              3. Create EstimateEvent
      |                                 type: JobCreatedFromEstimate
      |                                 payload: {jobId, jobNumber}
      |                                        |
```

### Implementation

#### 1. New gRPC Method in EstimatesService

**File:** `Src/Tofu.Invoices/EstimatesService.cs`

```csharp
public async Task NotifyJobCreatedFromEstimate(
    string accountId,
    string estimateId,
    string jobId,
    string jobNumber,
    string traceId,
    CancellationToken ct)
{
    var request = new NotifyJobCreatedRequest
    {
        AccountId = accountId,
        EstimateId = estimateId,
        JobId = jobId,
        JobNumber = jobNumber
    };

    await _estimatesApiClient.NotifyJobCreatedAsync(
        request,
        GetMetadata(traceId, accountId),
        cancellationToken: ct);
}
```

#### 2. Call from CreateJobFromEstimateCommandHandler

**File:** `Src/Jobs/Jobs.Application/Commands/CreateJobFromEstimateCommandHandler.cs`

Add after job is created and saved:

```csharp
public async Task<CreateJobFromEstimateResult> Handle(
    CreateJobFromEstimateCommand command,
    CancellationToken ct)
{
    // ... existing job creation logic ...

    await _jobsRepository.Insert(job, ct);
    await _jobDomainEventService.Save(job, ct);
    await _jobEstimateService.TryLinkEstimate(job, command.EstimateId, ct);

    // NEW: Notify Tofu.Invoices to create estimate event
    await _estimatesService.NotifyJobCreatedFromEstimate(
        command.AccountId,
        command.EstimateId,
        job.Id.ToString(),
        job.Number.ToString(),
        traceId,
        ct);

    return new CreateJobFromEstimateResult(job.ToDto());
}
```

### Required Changes in Tofu.Invoices

| Component | Change |
|-----------|--------|
| Proto file | Add `NotifyJobCreated` RPC method |

## Part 2: Invoice Link in Estimate Response

### Solution

Resolve invoice link at API layer when fetching estimate details. This avoids Tofu.Invoices changes and keeps the query simple.

### Implementation

#### 1. Update EstimateDto

**File:** `Src/Invoices.Api/Models/EstimateDto.cs`

```csharp
public class EstimateDto
{
    // ... existing fields ...

    public string? JobId { get; set; }

    /// <summary>Invoice ID if this estimate's job has an invoice.</summary>
    public string? InvoiceId { get; set; }  // NEW

    /// <summary>Invoice number for display.</summary>
    public string? InvoiceNumber { get; set; }  // NEW
}
```

#### 2. Update EstimatesController

**File:** `Src/Invoices.Api/Controllers/EstimatesController.cs`

```csharp
[HttpGet("{id}")]
public async Task<EstimateDto> GetEstimate(string id, CancellationToken ct)
{
    var estimate = await _estimatesService.Get(AccountId, id, RequestId, ct);
    var dto = estimate.MapToDto();

    // NEW: Resolve invoice link if estimate has job
    if (!string.IsNullOrEmpty(estimate.JobId))
    {
        dto = await EnrichWithInvoiceLink(dto, estimate.JobId, ct);
    }

    return dto;
}

private async Task<EstimateDto> EnrichWithInvoiceLink(
    EstimateDto dto,
    string jobId,
    CancellationToken ct)
{
    if (!Guid.TryParse(jobId, out var jobGuid))
        return dto;

    try
    {
        var query = new GetJobByIdQuery(AccountId, jobGuid);
        var result = await _dispatcher.DispatchQuery<GetJobByIdQuery, GetJobByIdResult>(query, ct);

        if (!string.IsNullOrEmpty(result?.Job?.InvoiceId))
        {
            dto.InvoiceId = result.Job.InvoiceId;

            // Fetch invoice number for display
            var invoice = await _invoicesService.Find(AccountId, result.Job.InvoiceId, RequestId, ct);
            dto.InvoiceNumber = invoice?.Number;
        }
    }
    catch (Exception ex)
    {
        _logger.LogWarning(ex, "Failed to resolve invoice link for estimate {EstimateId}", dto.Id);
        // Continue without invoice link - non-critical
    }

    return dto;
}
```

#### 3. Inject Dependencies

Ensure `EstimatesController` has:
- `IHandlerDispatcher _dispatcher` (for job query)
- `IInvoicesService _invoicesService` (for invoice lookup)

## API Changes Summary

| Endpoint | Change |
|----------|--------|
| `GET /estimates/{id}` | Response includes `invoiceId` and `invoiceNumber` when estimate's job has invoice |

---

## Data Flow

```
GET /estimates/{id}
  │
  ├─ Fetch estimate from Tofu.Invoices
  │
  ├─ Has JobId?
  │    ├─ No  → Return estimate (invoiceId = null)
  │    │
  │    └─ Yes → Query GetJobById
  │              │
  │              ├─ Job has InvoiceId?
  │              │    ├─ No  → Return estimate (invoiceId = null)
  │              │    │
  │              │    └─ Yes → Fetch invoice for number
  │              │              │
  │              │              └─ Return estimate with invoiceId + invoiceNumber
```
