# 4.1 Activity Phrasing Updates

## Overview

Update job activity event payloads per product specification.

**Source of truth**: `features/jobs/Job Timeline Events.csv`
**Activity Phrasing**: See `features/jobs/Activity.md`

### Scope

**In scope**:
- `visitCount` field for visit creation (1 = first visit, 2+ = subsequent)
- `workerName` field for visit events (null for now, workers service integration later)

**Out of scope**:
- `visitWorkerChanged` event - deferred
