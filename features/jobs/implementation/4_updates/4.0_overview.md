# Stage 4: Jobs Updates

## Overview

Stage 4 focuses on syncing job activity events with updated requirements, estimate-job integration improvements, sync API enhancements, and event enrichment.

**Source**: `Job_activity_new.pdf` (Ekaterina Lazukina)

## Goals

1. Update activity event phrasing to match new specification
2. Add missing activity events (finish visit)
3. Fix duplicate event issues (convert estimate to invoice)
4. Add estimate activity event when job is created from estimate
5. Enhance job sync endpoint with optional client data
6. Enrich job events with worker name for display

## Scope

### In Scope

- Activity event phrasing updates
- New activity events (finish visit)
- Fix duplicate events
- Estimate event when job created from estimate
- Invoice link in estimate response
- Include client info flag on job sync endpoint
- Worker name in job event payloads

### Out of Scope

- Job status model changes (4-status model vs current 7-status)
- Billing status as separate field
- Worker app visit actions (iteration 2)
- Undo functionality (separate iteration)
- Client archiving (moved to [Step 8](../8_client_archiving/overview.md))

## Implementation Order

1. **4.1 Activity Phrasing** - Update event messages and add missing events
2. **4.2 Event Worker Enrichment** - Add workerName to event payloads
3. **4.3 Amount Display Rules** - Stats and list amount display logic
4. **4.5 Create Job from Estimate** - Estimate event and invoice linking
5. **4.6 Sync Include Client** - Optional client data in sync response
6. **4.7 Jobs Without Client** - Allow job creation when client not yet synced
7. **4.8 DocNumber Enrichment** - Add document number to invoice/estimate event payloads
